Progreso del Build — desde 05 Mayo 2026

FIX IDL faltante corregido — app/public/idl/ocean_sense_pay.json creado manualmente. Sin este archivo, program era siempre null y todas las transacciones retornaban silenciosamente. Era el bloqueador principal.
FIX PROGRAM_ID incorrecto corregido — El fallback en useOceanSense.ts usaba un ID de ejemplo de Anchor (Fg6Pa…). Ahora apunta al ID real del programa: EawytSi….
FIX Seeds de getMintConfigPda corregidos — TypeScript usaba solo [Buffer.from("mint_config")]; el programa Rust requiere [b"mint_config", usdc_mint.key()]. El bug rompía todas las operaciones cPEN (claim, mint, redeem).
MITIG Oracle PEN/USD — mitigación frontend — Hook useExchangeRate que consume la API Frankfurter (USD/PEN en tiempo real, caché localStorage 1h). El tipo de cambio en la UI es dinámico. El on-chain rate sigue hardcodeado (3.80) — mitigación completa requiere Pyth.
NUEVO @oceansense/sdk — Cliente TypeScript framework-agnostic (sdk/src/client.ts). Expone registerBuoy, submitReading, claimRewardAsCpen, fetchBuoys, fetchCpenStats. IDL embebido, sin archivos externos. Compatible con Node.js, browser y cualquier framework.
NUEVO IoT Gateway — Servidor HTTP (gateway/index.ts) que recibe datos de sensores vía POST /reading y firma/envía la transacción Solana. Un ESP32 solo necesita hacer una llamada HTTP. Endpoints: /register · /reading · /buoys · /health.
NUEVO BUOY_SPEC.md — Especificaciones técnicas completas del prototipo físico: 5 sensores (temperatura, salinidad, oleaje, contaminación, GPS), arquitectura ESP32-S3 + ATECC608B + Raspberry Pi Zero 2W + SIM7600, sistema de energía solar, chasis IP68, conectividad 4G/LoRaWAN y hoja de ruta hardware v0.1 → v1.0.
NUEVO README.md actualizado — Documentación completa con instrucciones del SDK, ejemplo Arduino para ESP32, tabla de endpoints del gateway, variables de entorno separadas por módulo, y roadmap actualizado con los ítems completados.

¿Por qué Solana? — Reforzado por el build

🔒 Inmutabilidad de datos ambientales: Cada lectura del ESP32 llega firmada por el ATECC608B y registrada on-chain. Organismos como IMARPE, aseguradoras o la Armada Peruana pueden verificar datos sin intermediarios.
💰 Micropagos automáticos: El gateway firma y paga al operador sin intervención humana. Sin blockchain no hay forma de pagar a 40,000 pescadores en tiempo real sin una empresa central.
🔓 SDK abierto: Cualquier desarrollador puede integrar datos de Ocean-Sense con 3 líneas de TypeScript vía @oceansense/sdk — seguros paramétricos, apps de pesca, dashboards gubernamentales.

✓ Veredicto reforzado: Crypto sigue siendo necesario, no ornamental.
