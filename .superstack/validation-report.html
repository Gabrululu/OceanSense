Señales de Demanda (5 señales — score: 2/3 Moderado-Fuerte)

  • 💥 El Niño 2023-2024 causó $3B en pérdidas económicas en Perú por falta de monitoreo oceánico descentralizado — el problema tiene un costo documentado y reciente.
  • 🎣 40,000 pescadores artesanales afectados por la ausencia de datos de temperatura, corrientes y contaminación en tiempo real a lo largo de 3,080 km de litoral.
  • 🏛️ IMARPE y SENAHMI tienen cobertura documentadamente insuficiente — sistemas centralizados sin incentivos para participación comunitaria. Hay un vacío institucional real.
  • 🌦️ WeatherXM prueba el modelo: comunidad de estaciones meteorológicas descentralizadas recompensadas por datos ambientales tiene tracción en Web3. OceanSense es el análogo para datos oceánicos.
  • 🔗 Solana lidera DePIN: Helium (>500k dispositivos), Hivemapper, GEODNET, Render demuestran que el modelo de incentivos on-chain para hardware funciona a escala.

¿Qué se rompe si quitas el blockchain?

🔒 Integridad de datos: Sin blockchain, los datos oceánicos pueden ser falsificados. Organismos como IMARPE o aseguradoras no confiarían en lecturas no inmutables.
💰 Capa de incentivos: Sin token, no hay forma de pagar a pescadores artesanales en tiempo real sin una empresa central financiando cada pago.
🔓 Composabilidad: Otros protocolos (seguros paramétricos, mercados de datos) pueden leer los datos on-chain sin permisos — esto no existe en un sistema centralizado.

✓ Veredicto: Crypto es necesario, no ornamental.

Mapa de Riesgos

⚡ Hardware / Bootstrap Alto
La integración IoT real (ESP32 + sensores CTD) es trabajo futuro. El MVP on-chain está construido pero sin hardware físico, la red no puede operar. Además: ¿quién financia el vault USDC inicial? Una red de 100 boyas submitting cada hora puede requerir $2,400–12,000/día en recompensas.
⚠ Regulatorio (cPEN) Medio
cPEN es un synthetic del Sol Peruano (PEN). Operar una stablecoin pegged a moneda local puede requerir análisis con la SBS/SUNAT. En v1, reemplazar cPEN por USDC puro eliminaría este riesgo mientras se valida el producto.
⚠ UX / Adopción Medio
Los pescadores artesanales necesitan conectividad, smartphone, y wallet de Solana. La curva de onboarding es alta. Considerar una interfaz de operador simplificada (SMS, app nativa) o intermediarios (cooperativas pesqueras).
⚠ Oracle PEN/USD Medio
La tasa 1 USDC = 3.80 cPEN está hardcodeada. En producción, el tipo de cambio PEN/USD varía. Sin un oracle en tiempo real (Pyth tiene feeds de fiat), el peg puede desviarse causando pérdidas para operadores o el protocolo.

Integración vs. Desarrollo Propio

El programa on-chain personalizado es la decisión correcta. La lógica de incentivos DePIN, el vault colateralizado y el token cPEN requieren lógica on-chain propia — no hay protocolo existente que cubra exactamente este caso de uso.


Integraciones recomendadas para acelerar:

  • Pyth Network — Oracle PEN/USD para el peg dinámico de cPEN (en lugar del rate hardcodeado)
  • Token-2022 — Ya integrado correctamente con Transfer Fee + Metadata + Freeze Authority
  • Marinade/Jito (futuro) — Rendimiento sobre el colateral USDC del vault cuando esté inactivo

Próximos Pasos Concretos

  • 1
    Validación de UX con usuarios reales: conseguir 3-5 pescadores o técnicos de cooperativas pesqueras para probar el flujo completo con datos mock antes de invertir en hardware.
  • 2
    Definir modelo de financiamiento del vault: evaluar grant de Solana Foundation (DePIN/DeSci track), co-inversión con aseguradoras pesqueras, o presupuesto de PRODUCE/FONDEPES.
  • 3
    Consulta legal sobre cPEN: abogado fintech peruano con experiencia en SBS. Si hay riesgo, considerar lanzar v1 solo con recompensas en USDC y agregar cPEN cuando haya claridad regulatoria.
  • 4
    Integrar Pyth para oracle PEN/USD: reemplazar el rate hardcodeado de 3.80 por un oracle en tiempo real. Pyth tiene feeds para USD/PEN.
  • 5
    Prototipo hardware mínimo: ESP32 + sensor de temperatura DS18B20 + SIM 4G (~$40 de materiales). Suficiente para una primera prueba de concepto real en el litoral.
  • 6
    Canal B2G: presentar datos como complemento a IMARPE/PRODUCE. Organismos públicos pueden convertirse en financiadores del vault a cambio de acceso prioritario a datos.
  • 7
    Registrar en Colosseum Hackathon: la categoría DePIN/DeSci tiene tracción histórica. Usar /colosseum-copilot para ver proyectos similares que ganaron y qué diferenciaba a los ganadores.
